#  ShoppingMall – Java Desktop Application

این پروژه یک **سیستم فروشگاه (Shopping Mall)** به زبان **Java** است که با استفاده از **Swing** برای رابط گرافیکی و **Maven** برای مدیریت وابستگی‌ها پیاده‌سازی شده است.  
پروژه شامل دو پنل مجزا برای **مدیر (Admin)** و **مشتری (Customer)** می‌باشد. 
این پروژه به عنوان یک پروژه دانشجویی برای درس برنامه‌نویسی پیشرفته طراحی شده و هدف آن تمرین مفاهیم OOP، کار با Swing، و تفکیک لایه‌های برنامه می‌باشد.

##  ویژگی‌های پروژه

###  بخش مدیر (Admin)
- مشاهده لیست محصولات
- افزودن محصول جدید
- ویرایش قیمت و موجودی محصولات
- حذف محصول
- مشاهده جزئیات هر محصول
- رابط گرافیکی مدرن با FlatLaf

###  بخش مشتری (Customer)
- مشاهده محصولات فروشگاه
- جستجوی محصول
- مرتب‌سازی محصولات (قیمت، زمان اضافه شدن، موجودی)
- افزودن محصول به سبد خرید
- مشاهده سبد خرید
## ساختار پروژه
- معماری لایه‌ای:
  - `model`
  - `repository`
  - `service`
  - `gui`

این بخش مهم‌ترین بخش پروژه محسوب می‌شود. model همه‌ی ویژگی‌های مد نظر و همه‌ی کارهای برنامه‌ی ما را نشان می‌دهد که قصد انجام آن‌ها را داریم. این پکیج شامل ۵ کلاس user, admin, customer, product, cart می‌باشد که ادمین و کاستومر از یوزر ارث‌بری کرده‌اند. می‌توان گفت این پایه‌ی اصلی پروژه است که کدهای بقیه‌ی قسمت‌ها به این قسمت وابسته بوده و روی آن سوار می‌شوند. repository به نوعی زیرمجموعه‌ی model محسوب می‌شود. این قسمت دیتابیس ما و همه‌ی شی‌هایی که قرار است از کلاس‌های مدل ساخته شوند را در خود نگه می‌دارد و ذخیره می‌کند؛ به نحوی که پس از بستن و باز کردن برنامه، ما به اطلاعاتی که قبلا وارد کرده بودیم دسترسی داریم. برای این قسمت ما ابتدا ریپازیتوری‌های لازم cart, product و customer را به صورت in-memory و داخل برنامه‌ای پیاده‌سازی کردیم و سپس با استفاده از کتابخانه‌ی GSON آن را به صورت دیتابیس دائمی دراوردیم. بعد از آن به سراغ پکیج services رفتیم که به نوعی به هر دو پکیج model و repository وابستگی دارد. این پکیج عملیات مختلفی را روی ریپازیتوری‌ها انجام می‌دهد و ما برای ارتباط گیری با ریپازیتوری و خواندن و نوشتن بر روی ریپو از کلاس‌های این پکیج استفاده می‌کنیم. در نهایت، پکیج رابط گرافیکی یا gui را پیاده سازی کردیم که با استفاده از سه پکیج قبلی، نیاز ما را که عملیات فروشگاهی بود را براورده می‌کند. در این قسمت ما سه کلاس Entry, Display, ProductView را مشترکا هم برای ادمین و هم برای کاستومر تعریف کردیم. کلاس entry پنلی است که هر چه را یوزر هنگام ورود به برنامه باید ببیند فراهم می‌کند. در این کلاس ما  منطقی را اجرا نمی‌کنیم. کلاس ProductView صرفا مشخص می‌کند که شی مشترک پروداکت برای مشتری و برای ادمین چگونه باید نمایش داده شوند. هیچ منطق و استفاده از ریپازیتوری در این قسمت وجود ندارد. کلاس Display در واقع منطق پشت کلاس‌های نمایشی  ProductView و Entry را به اجرا در اورده و همچنین پنل‌های اختصاصی تعریف شده برای هر نقش - مانند shoppingCartPanel برای customer و AdminAddProduct , AdminEditProduct, AdminViewDetailsبرای admin - را یکپارچه می‌کند. کلاس LoginDialog پنل لاگین را حین شروع برنامه به نمایش در میاورد. وظیفه‌ی این کلاس این است که یوزرنیم و پسورد را گرفته و مشخص کند که کاربر، چه نقشی دارد و با توجه به آن، پنل CustomerEntryPanel یا AdminEntryPanel را به او نشان می‌دهد.
